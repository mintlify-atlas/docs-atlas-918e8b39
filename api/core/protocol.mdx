The `Protocol` class implements the Model Context Protocol framing layer on top of a pluggable transport. It handles JSON-RPC message routing, request/response correlation, notifications, and progress tracking.

## Package

```kotlin
io.modelcontextprotocol.kotlin.sdk.shared
```

## Class Definition

```kotlin
abstract class Protocol(internal val options: ProtocolOptions?)
```

The Protocol class is abstract and typically used through `Server` or `Client` implementations rather than directly.

### request

Sends a request and waits for a response.

```kotlin
suspend fun <T : RequestResult> request(
    request: Request,
    options: RequestOptions? = null
): T
```

<ParamField path="request" type="Request" required>
  The request to send
</ParamField>

<ParamField path="options" type="RequestOptions">
  Optional request options (timeout, progress callback, etc.)
</ParamField>

**Returns:** The typed response result

**Throws:** `McpException` on error or timeout

**Example:**

```kotlin
val result: ListToolsResult = protocol.request(
    ListToolsRequest(),
    RequestOptions(timeout = 30.seconds)
)
```

<ParamField path="method" type="Method" required>

### onClose

Callback invoked when the connection closes.

```kotlin
open fun onClose()
```

Override this method to perform cleanup when the connection closes.

**Example:**

```kotlin
class MyProtocol : Protocol(options) {
    override fun onClose() {
        println("Connection closed")
        cleanup()
    }
}
```

### onError

Callback invoked when an error occurs.

```kotlin
open fun onError(error: Throwable)
```

<ParamField path="error" type="Throwable" required>
  The error that occurred
</ParamField>

Override this method to handle errors. Note that errors are not necessarily fatal.

**Example:**

## Progress Tracking

The protocol automatically handles progress notifications. When you send a request with an `onProgress` callback:

1. The protocol registers a progress handler for the request ID
2. It adds a `progressToken` to the request parameters
3. When the remote side sends progress notifications, your callback is invoked
4. The progress handler is cleaned up when the request completes

```kotlin
val result = protocol.request(
    CallToolRequest(params),
    RequestOptions(
        onProgress = { progress ->
            updateProgressBar(progress.progress, progress.total)
        }
    )
)
```

## Error Handling

The protocol handles errors at multiple levels:

1. **Transport errors** - Reported via `onError()`, connection closed
2. **Protocol errors** - Invalid messages, serialization failures
3. **Request errors** - Remote side returns error response (throws `McpException`)
4. **Timeouts** - Request exceeds timeout (throws `TimeoutCancellationException`)

```kotlin
try {
    val result = protocol.request(request, options)
} catch (e: McpException) {
    when (e.code) {
        RPCError.ErrorCode.METHOD_NOT_FOUND -> println("Method not supported")
        RPCError.ErrorCode.INVALID_PARAMS -> println("Invalid parameters")
        else -> println("Error: ${e.message}")
    }
} catch (e: TimeoutCancellationException) {
    println("Request timed out")
}
```
