The `ServerSession` class represents a single client connection to an MCP server. Each session manages its own protocol state, client capabilities, and request handlers.

## Package

```kotlin
io.modelcontextprotocol.kotlin.sdk.server
```

```kotlin
    protected val instructions: String?
```

## Methods

### Lifecycle Callbacks

#### onInitialized

Registers a callback invoked when the session has completed initialization.

```kotlin
fun onInitialized(block: () -> Unit)
```

**Example:**

```kotlin
session.onInitialized {
    println("Session ${session.sessionId} initialized")
    println("Client: ${session.clientVersion?.name} ${session.clientVersion?.version}")
}
```

#### onClose

Registers a callback invoked when the session is closing.

```kotlin
fun onClose(block: () -> Unit)
```

**Example:**

```kotlin
session.onClose {
    println("Session ${session.sessionId} closed")
    cleanupSessionResources()
}
```

### Client Requests

These methods allow the server to make requests to the connected client.

#### ping

Sends a ping request to check connectivity.

```kotlin

```

**Example:**

```kotlin
try {
    session.ping()
    println("Client is responsive")
} catch (e: Exception) {
    println("Client not responding")
}
```

#### createMessage

Creates a message using the client's sampling capability (if supported).

```kotlin
suspend fun createMessage(
    params: CreateMessageRequest,
    options: RequestOptions? = null
): CreateMessageResult
```

<ParamField path="params" type="CreateMessageRequest" required>
  Parameters for message creation
</ParamField>

<ParamField path="options" type="RequestOptions">
  Optional request options
</ParamField>

**Example:**

```kotlin

    CreateMessageRequest(
        CreateMessageRequestParams(
            messages = listOf(
                SamplingMessage(
                    role = Role.User,
                    content = TextContent("Summarize this document")

            maxTokens = 1000
        )
    )

```

#### listRoots

Lists the available roots from the client's perspective (if supported).

```kotlin
suspend fun listRoots(
    params: JsonObject = EmptyJsonObject,
    options: RequestOptions? = null
): ListRootsResult
```

**Example:**

```kotlin
if (session.clientCapabilities?.roots != null) {
    val roots = session.listRoots()
    roots.roots.forEach { root ->
        println("Root: ${root.name} (${root.uri})")
    }
}
```

Sends a message to the client requesting an elicitation (form display).

```kotlin
suspend fun createElicitation(
    message: String,
    requestedSchema: ElicitRequestParams.RequestedSchema,
    options: RequestOptions? = null
): ElicitResult
```

<ParamField path="message" type="String" required>
  Message to display to the end user
</ParamField>

<ParamField path="requestedSchema" type="RequestedSchema" required>
  Schema for the elicitation result
</ParamField>

**Example:**

```kotlin
if (session.clientCapabilities?.elicitation != null) {
    val result = session.createElicitation(
        message = "Please provide your preferences",
        requestedSchema = ElicitRequestParams.RequestedSchema(

            properties = buildJsonObject {
                put("theme", buildJsonObject {
                    put("type", "string")
                    put("enum", buildJsonArray {
                        add("light")
                        add("dark")
```

## ClientConnection

The `clientConnection` property provides access to client-facing operations:

```kotlin
interface ClientConnection {
    suspend fun ping(): EmptyResult
    suspend fun createMessage(params: CreateMessageRequest, options: RequestOptions? = null): CreateMessageResult
    suspend fun listRoots(params: JsonObject = EmptyJsonObject, options: RequestOptions? = null): ListRootsResult
    suspend fun createElicitation(message: String, requestedSchema: ElicitRequestParams.RequestedSchema, options: RequestOptions? = null): ElicitResult
    suspend fun sendLoggingMessage(notification: LoggingMessageNotification)
    suspend fun sendResourceUpdated(notification: ResourceUpdatedNotification)
    suspend fun sendResourceListChanged()
    suspend fun sendToolListChanged()
    suspend fun sendPromptListChanged()
}
```

**Example:**

```kotlin
val connection = session.clientConnection
connection.sendLoggingMessage(
    LoggingMessageNotification(
        LoggingMessageNotificationParams(
            level = LoggingLevel.Debug,
            logger = "server",
            data = "Debug info"
        )
    )
)
```

## Session Lifecycle

A typical session lifecycle:

```kotlin
val server = Server(serverInfo, options)

// 1. Create session with transport
val session = server.createSession(transport)

// 2. Session automatically handles initialization
//    - Receives InitializeRequest from client
//    - Responds with InitializeResult (capabilities, version)
//    - Waits for InitializedNotification

// 3. Callbacks are invoked
session.onInitialized {
    println("Client initialized: ${session.clientCapabilities}")
}

// 4. Session handles client requests
//    - Tools, prompts, resources based on server capabilities

// 5. Server can make requests to client
val roots = session.listRoots()

// 6. Session closes
session.close()
```

## Capability Checking

The session automatically enforces capability checks:

```kotlin
// These methods throw IllegalStateException if client doesn't support the capability
try {
    session.createMessage(params)  // Requires sampling capability
} catch (e: IllegalStateException) {
    println("Client does not support sampling")
}

try {
    session.listRoots()  // Requires roots capability
} catch (e: IllegalStateException) {
    println("Client does not support roots")
}
```

You can check capabilities manually:

```kotlin
if (session.clientCapabilities?.sampling != null) {
    val result = session.createMessage(params)
}

if (session.clientCapabilities?.roots != null) {
    val roots = session.listRoots()
}

if (session.clientCapabilities?.elicitation != null) {
    val elicitation = session.createElicitation(message, schema)
}
```

## Logging Levels

The session tracks the client's logging level preference:

```kotlin
// Client sets logging level via SetLevelRequest
// Session automatically filters outgoing log messages

session.sendLoggingMessage(
    LoggingMessageNotification(
        LoggingMessageNotificationParams(
            level = LoggingLevel.Debug,  // Only sent if client level allows
            logger = "server",
            data = "Debug message"
        )
    )
)
```

Logging levels (from least to most verbose):
- `LoggingLevel.Critical`
- `LoggingLevel.Error`
- `LoggingLevel.Warning`
- `LoggingLevel.Info`
- `LoggingLevel.Debug`

## Complete Example

```kotlin
val server = Server(serverInfo, options)

val session = server.createSession(transport)

session.onInitialized {
    println("Client connected: ${session.clientVersion?.name}")
    
    // Check client capabilities
    session.clientCapabilities?.let { caps ->
        if (caps.roots != null) {
            println("Client supports roots")
        }
        if (caps.sampling != null) {
            println("Client supports sampling")
        }
    }
}

session.onClose {
    println("Session ${session.sessionId} closed")
}

// Server can log to client
session.sendLoggingMessage(
    LoggingMessageNotification(
        LoggingMessageNotificationParams(
            level = LoggingLevel.Info,
            logger = "my-server",
            data = "Server ready"
        )
    )
)

// Make requests to client if supported
if (session.clientCapabilities?.roots != null) {
    val roots = session.listRoots()
    println("Client roots: ${roots.roots}")
}
```
