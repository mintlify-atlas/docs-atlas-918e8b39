The WebSocket transport provides full-duplex, bidirectional communication with low latency. It's ideal for scenarios requiring lots of server-initiated notifications or long-running interactive sessions.

## Overview

WebSocket transport features:

- Full-duplex bidirectional communication
- Low latency for real-time interactions
- Binary or text frame support
- Automatic connection management
- Works well behind reverse proxies

## Client Setup

### Basic Example

```kotlin
import io.ktor.client.HttpClient
import io.ktor.client.plugins.websocket.WebSockets
import io.modelcontextprotocol.kotlin.sdk.client.Client
import io.modelcontextprotocol.kotlin.sdk.client.WebSocketClientTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation

val httpClient = HttpClient {
    install(WebSockets)
}

val transport = WebSocketClientTransport(
    client = httpClient,
    urlString = "ws://localhost:3000/mcp"
)

val client = Client(
    clientInfo = Implementation(
        name = "my-client",
        version = "1.0.0"
    )
)

client.connect(transport)
```

### With TLS/WSS

```kotlin
val transport = WebSocketClientTransport(

```

### Complete Client Example

```kotlin
import io.ktor.client.HttpClient
import io.ktor.client.plugins.websocket.WebSockets
import io.modelcontextprotocol.kotlin.sdk.client.Client
import io.modelcontextprotocol.kotlin.sdk.client.WebSocketClientTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import kotlinx.coroutines.runBlocking

fun main() = runBlocking {
    val httpClient = HttpClient {
        install(WebSockets) {
            pingInterval = 30_000  // 30 seconds
        }
    }

    val client = Client(
        clientInfo = Implementation(
            name = "example-client",
            version = "1.0.0"
        )
    )

    val transport = WebSocketClientTransport(
        client = httpClient,
        urlString = "ws://localhost:3000/mcp"
    )

    client.connect(transport)

    // List available resources
    val resources = client.listResources()
    println("Available resources: ${resources.resources.map { it.name }}")

    // Clean up
    client.close()
    httpClient.close()
}
```

## Server Setup

### Basic Example

```kotlin
import io.ktor.server.cio.CIO
import io.ktor.server.engine.embeddedServer
import io.ktor.server.routing.routing
import io.ktor.server.websocket.WebSockets
import io.ktor.server.websocket.webSocket
import io.modelcontextprotocol.kotlin.sdk.server.Server
import io.modelcontextprotocol.kotlin.sdk.server.ServerOptions
import io.modelcontextprotocol.kotlin.sdk.server.WebSocketMcpServerTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import io.modelcontextprotocol.kotlin.sdk.types.ServerCapabilities

fun main() {
    val server = Server(
        serverInfo = Implementation(
            name = "my-server",
            version = "1.0.0"
        ),
        options = ServerOptions(
            capabilities = ServerCapabilities(
                tools = ServerCapabilities.Tools(listChanged = true)
            )
        )
    )

    embeddedServer(CIO, port = 3000) {
        install(WebSockets)
        
        routing {
            webSocket("/mcp") {
                val transport = WebSocketMcpServerTransport(this)
                server.createSession(transport)
            }
        }
    }.start(wait = true)
}
```

## Configuration Options

<Tabs>
  <Tab title="Client">
    ### WebSocketClientTransport Parameters

    - **client** (required): `HttpClient` - Ktor HTTP client with WebSockets plugin
    - **urlString** (optional): `String?` - WebSocket URL (null to use request builder)
    - **requestBuilder** (optional): `HttpRequestBuilder.() -> Unit` - Custom request config

    ### WebSocket Client Configuration

    ```kotlin
    HttpClient {
        install(WebSockets) {
            pingInterval = 30_000      // Ping interval in milliseconds
            maxFrameSize = 1024 * 1024 // Max frame size in bytes
            contentConverter = ...      // Custom content converter
        }
    }
    ```
  </Tab>
  
  <Tab title="Server">
    ### WebSocketMcpServerTransport Parameters

    - **session** (required): `WebSocketServerSession` - Ktor WebSocket session

    ### WebSocket Server Configuration

    ```kotlin
    install(WebSockets) {
        pingPeriod = Duration.seconds(30)   // Ping interval
        timeout = Duration.seconds(60)      // Connection timeout
        maxFrameSize = Long.MAX_VALUE       // Max frame size
        masking = false                     // WebSocket masking
    }
    ```

## Subprotocol

The MCP WebSocket transport uses the `mcp` subprotocol:

```kotlin
// Client automatically sets
header(HttpHeaders.SecWebSocketProtocol, "mcp")

// Server validates
val subprotocol = session.call.request.headers[HttpHeaders.SecWebSocketProtocol]
check(subprotocol == "mcp") { "Invalid subprotocol" }
```

## Complete Example

<CodeGroup>
```kotlin Server
import io.ktor.server.cio.CIO
import io.ktor.server.engine.embeddedServer
import io.ktor.server.routing.routing
import io.ktor.server.websocket.WebSockets
import io.ktor.server.websocket.webSocket
import io.modelcontextprotocol.kotlin.sdk.server.Server
import io.modelcontextprotocol.kotlin.sdk.server.ServerOptions
import io.modelcontextprotocol.kotlin.sdk.server.WebSocketMcpServerTransport
import io.modelcontextprotocol.kotlin.sdk.types.CallToolResult
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import io.modelcontextprotocol.kotlin.sdk.types.ServerCapabilities
import io.modelcontextprotocol.kotlin.sdk.types.TextContent
import kotlin.time.Duration.Companion.seconds

fun main() {
    val server = Server(
        serverInfo = Implementation(
            name = "ws-server",
            version = "1.0.0"
        ),
        options = ServerOptions(
            capabilities = ServerCapabilities(
                tools = ServerCapabilities.Tools(listChanged = true)
            )
        )
    )

    // Add a tool
    server.addTool(
        name = "greet",
        description = "Greets a person by name"
    ) { request ->
        val name = request.arguments?.get("name")?.toString() ?: "stranger"
        CallToolResult(
            content = listOf(TextContent("Hello, $name!"))
        )
    }

    embeddedServer(CIO, host = "127.0.0.1", port = 3000) {
        install(WebSockets) {
            pingPeriod = 30.seconds
            timeout = 60.seconds
        }
        
        routing {
            webSocket("/mcp") {
                val transport = WebSocketMcpServerTransport(this)
                val session = server.createSession(transport)
                
                session.onClose {
                    println("WebSocket session closed")
                }
            }
        }
    }.start(wait = true)
}
```

```kotlin Client
import io.ktor.client.HttpClient
import io.ktor.client.plugins.websocket.WebSockets
import io.modelcontextprotocol.kotlin.sdk.client.Client
import io.modelcontextprotocol.kotlin.sdk.client.WebSocketClientTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import kotlinx.coroutines.runBlocking
import kotlinx.serialization.json.buildJsonObject
import kotlinx.serialization.json.put

fun main() = runBlocking {
    val httpClient = HttpClient {
        install(WebSockets) {
            pingInterval = 30_000
        }
    }

    val client = Client(
        clientInfo = Implementation(
            name = "ws-client",
            version = "1.0.0"
        )
    )

    val transport = WebSocketClientTransport(
        client = httpClient,
        urlString = "ws://localhost:3000/mcp"
    )

    client.connect(transport)

    // Call the greet tool
    val result = client.callTool(
        name = "greet",
        arguments = buildJsonObject {
            put("name", "Alice")
        }
    )

    println(result.content.joinToString())

    client.close()
    httpClient.close()
}
```

## Best Practices

1. **Enable ping/pong**: Keep connections alive through firewalls and proxies
2. **Set reasonable timeouts**: Configure both ping and connection timeouts
3. **Handle reconnection**: Implement exponential backoff for automatic reconnection
4. **Limit frame size**: Prevent memory exhaustion from large frames
5. **Use WSS in production**: Always use secure WebSockets over TLS
6. **Validate subprotocol**: Ensure the `mcp` subprotocol is used
7. **Clean up resources**: Properly close connections and free resources
