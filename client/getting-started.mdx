## Overview

The MCP Kotlin SDK client enables Kotlin applications to connect to MCP servers and interact with their resources, prompts, and tools. This guide walks you through creating your first client.

## Installation

<Steps>
  <Step title="Add the Maven Central repository">
    Add Maven Central to your `build.gradle.kts`:
    
    ```kotlin
    repositories {
        mavenCentral()
    }
    ```
  </Step>
  
  <Step title="Add the client dependency">
    Add the client SDK dependency:
    
    ```kotlin
    dependencies {
        implementation("io.modelcontextprotocol:kotlin-sdk-client:$mcpVersion")
    }
    ```
    
    <Note>
      If you need both client and server functionality, use `kotlin-sdk` instead of `kotlin-sdk-client`.
    </Note>
  </Step>
  
  <Step title="Add Ktor engine dependency">
    The SDK uses Ktor but doesn't include an engine by default. Add one:
    
    ```kotlin
    dependencies {
        implementation("io.ktor:ktor-client-cio:$ktorVersion")
    }
    ```
  </Step>
</Steps>

## Creating a Basic Client

<Steps>
  <Step title="Import required classes">
    ```kotlin
    import io.ktor.client.HttpClient
    import io.ktor.client.plugins.sse.SSE
    import io.modelcontextprotocol.kotlin.sdk.client.Client
    import io.modelcontextprotocol.kotlin.sdk.client.StreamableHttpClientTransport
    import io.modelcontextprotocol.kotlin.sdk.types.Implementation
    import kotlinx.coroutines.runBlocking
    ```
  </Step>
  
  <Step title="Initialize the client">
    Create a client instance with your application information:
    
    ```kotlin
    val client = Client(
        clientInfo = Implementation(
            name = "example-client",
            version = "1.0.0"
        )
    )
    ```
    
    <ParamField path="clientInfo" type="Implementation" required>
      Information about your client application
      
      <ParamField path="clientInfo.name" type="String" required>
        Name of your client application
      </ParamField>
      
      <ParamField path="clientInfo.version" type="String" required>
        Version of your client application
      </ParamField>
    </ParamField>
  </Step>
  
  <Step title="Create a transport">
    Set up a transport mechanism to communicate with the server:
    
    ```kotlin
    val httpClient = HttpClient { install(SSE) }
    
    val transport = StreamableHttpClientTransport(
        client = httpClient,
        url = "http://localhost:3000/mcp"
    )
    ```
    
    <Note>
      The SDK supports multiple transports: Streamable HTTP, STDIO, SSE, and WebSocket. Streamable HTTP is recommended for remote servers.
    </Note>
  </Step>
  
  <Step title="Connect to the server">
    Establish the connection (this performs the initialization handshake):
    
    ```kotlin
    client.connect(transport)
    ```
    
    After connecting, you can access server information:
    
    ```kotlin
    val capabilities = client.serverCapabilities
    val serverInfo = client.serverVersion
    ```
  </Step>
</Steps>

## Complete Example

Here's a complete example that connects to a server and lists available tools:

```kotlin
import io.ktor.client.HttpClient
import io.ktor.client.plugins.sse.SSE
import io.modelcontextprotocol.kotlin.sdk.client.Client
import io.modelcontextprotocol.kotlin.sdk.client.StreamableHttpClientTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import kotlinx.coroutines.runBlocking

fun main(args: Array<String>) = runBlocking {
    val url = args.firstOrNull() ?: "http://localhost:3000/mcp"

    val httpClient = HttpClient { install(SSE) }

    val client = Client(
        clientInfo = Implementation(
            name = "example-client",
            version = "1.0.0"
        )
    )

    val transport = StreamableHttpClientTransport(
        client = httpClient,
        url = url
    )

    // Connect to server
    client.connect(transport)

    // List available tools
    val tools = client.listTools().tools

    println(tools)
}
```

## Using STDIO Transport

For local process-based servers (like command-line tools), use STDIO transport:

```kotlin
import io.modelcontextprotocol.kotlin.sdk.client.Client
import io.modelcontextprotocol.kotlin.sdk.client.StdioClientTransport
import io.modelcontextprotocol.kotlin.sdk.types.Implementation
import kotlinx.io.asSink
import kotlinx.io.asSource
import kotlinx.io.buffered

val process = ProcessBuilder("path/to/server").start()

val transport = StdioClientTransport(
    input = process.inputStream.asSource().buffered(),
    output = process.outputStream.asSink().buffered(),
)

val client = Client(
    clientInfo = Implementation(name = "mcp-client", version = "1.0.0")
)

client.connect(transport)
```

## Client API Methods

Once connected, the client provides methods to interact with the server:

### Tools

```kotlin
// List all available tools
val toolsList = client.listTools()

// Call a tool
val result = client.callTool(
    name = "tool-name",
    arguments = mapOf("param" to "value")
)
```

### Resources

```kotlin
// List resources
val resources = client.listResources()

// Read a resource
val content = client.readResource(
    ReadResourceRequest(
        ReadResourceRequestParams(uri = "resource://example")
    )
)
```

### Prompts

```kotlin
// List prompts
val prompts = client.listPrompts()

// Get a specific prompt
val prompt = client.getPrompt(
    GetPromptRequest(
        GetPromptRequestParams(name = "prompt-name")
    )
)
```

## Client Configuration

Configure client capabilities and behavior with `ClientOptions`:

```kotlin
import io.modelcontextprotocol.kotlin.sdk.client.ClientOptions
import io.modelcontextprotocol.kotlin.sdk.types.ClientCapabilities

val client = Client(
    clientInfo = Implementation("demo-client", "1.0.0"),
    options = ClientOptions(
        capabilities = ClientCapabilities(
            roots = ClientCapabilities.Roots(listChanged = true),
            sampling = buildJsonObject { putJsonObject("tools") { } }
        ),
        enforceStrictCapabilities = true
    )
)
```

<ParamField path="options" type="ClientOptions">
  Configuration options for the client
  
  <ParamField path="options.capabilities" type="ClientCapabilities">
    Capabilities that this client supports (roots, sampling, elicitation)
  </ParamField>
  
  <ParamField path="options.enforceStrictCapabilities" type="Boolean" default="true">
    Whether to strictly enforce capabilities when interacting with the server
  </ParamField>
</ParamField>
