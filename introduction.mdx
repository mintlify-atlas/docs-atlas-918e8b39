The MCP Kotlin SDK is a comprehensive implementation of the [Model Context Protocol](https://modelcontextprotocol.io) that enables Kotlin applications to build MCP clients and servers across multiple platforms.

## What is MCP?

The Model Context Protocol (MCP) allows applications to provide context for Large Language Models (LLMs) in a standardized way, separating the concerns of providing context from the actual LLM interaction.

## MCP Primitives

The SDK implements core MCP primitives that enable communication between clients and servers:

| Primitive | Description | Server Role | Client Role |
|-----------|-------------|-------------|-------------|
| **Prompts** | Interactive templates for LLM interactions | Provides prompt templates with arguments | Requests and uses prompts |
| **Resources** | Contextual data sources | Exposes data (files, API responses, etc.) | Reads and subscribes to resources |
| **Tools** | Executable functions | Defines functions the LLM can invoke | Calls tools to perform actions |
| **Sampling** | Server-initiated LLM requests | Requests LLM completions from client | Executes LLM calls and returns results |

## Architecture

The SDK is organized into focused modules:

```
io.modelcontextprotocol:kotlin-sdk          # Umbrella (client + server)
io.modelcontextprotocol:kotlin-sdk-client   # Client-only APIs
io.modelcontextprotocol:kotlin-sdk-server   # Server-only APIs
io.modelcontextprotocol:kotlin-sdk-core     # Core types and protocol
```

<Note>
  Use the umbrella `kotlin-sdk` artifact if you need both client and server capabilities, or choose `kotlin-sdk-client` or `kotlin-sdk-server` for focused dependencies.
</Note>

## Platform Requirements

- **Kotlin**: 2.2 or higher
- **JVM**: Java 11 or higher (when targeting JVM)
- **Ktor**: Not included transitively - you must add Ktor client/server dependencies
